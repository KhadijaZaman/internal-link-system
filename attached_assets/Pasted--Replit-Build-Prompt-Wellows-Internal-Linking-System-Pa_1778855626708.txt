## Project: `wellows-internal-linking`

## The four background jobs (run via APScheduler)

All jobs run inside the FastAPI process. Use APScheduler's `AsyncIOScheduler` with cron triggers.

### Job 1 — `crawl_link_map` (Sunday 02:00 UTC)

Replaces n8n "WF2 — Link Map Builder". Chunked crawler.

**Logic:**

1. Fetch `SITEMAP_URL`. Parse all `<loc>` entries using `xml.etree.ElementTree` (handle both regular and sitemap-index formats — if it's an index, fetch each sub-sitemap).
2. Read `crawl_progress.last_offset` from DB. Slice URLs `[offset : offset + 200]`.
3. For each URL in the chunk (async, with `asyncio.Semaphore(5)` and 1-second polite delay):
   - GET the page with `httpx` (15s timeout, follow redirects, custom User-Agent: `WellowsLinkMapBot/1.0`)
   - Skip if status != 200 or content-type isn't HTML
   - Use `selectolax` (faster than BeautifulSoup) to parse:
     - `<title>` → save
     - `<h1>` first one → save
     - All `<a href>` where href resolves to the same domain
     - For each link: source_url, target_url (normalize relative paths, strip fragments, drop self-links), anchor text (max 150 chars), surrounding 80 chars of text
4. Upsert into `link_graph` (on conflict do nothing — keeps history clean)
5. After chunk completes:
   - Update `crawl_progress.last_offset += 200` (reset to 0 when reaching end of sitemap)
   - **Recompute `link_stats` for ALL urls** (this is fast, do it in a single SQL query):
     - `inbound_count = COUNT(*) FROM link_graph WHERE target_url = url`
     - `outbound_count = COUNT(*) FROM link_graph WHERE source_url = url`
     - `is_orphan = inbound_count == 0`
     - `is_dead_end = outbound_count == 0`
     - `internal_pagerank`: run 10 iterations of standard PageRank in Python (damping 0.85). For a 500-5000 page graph this takes < 1 second using a dict-based adjacency list.
6. Write to a snapshot JSON file in `/tmp/link_graph_export.json` for the dashboard to read

**Error handling:** Wrap each URL fetch in try/except. Log failures to a `crawl_errors` table or just stdout. Never let one bad page kill the whole job.

### Job 2 — `gsc_inventory_and_losers` (Monday 03:00 UTC)

Replaces n8n "WF3".

**Logic:**

1. Authenticate to GSC using stored refresh token. Build the Search Console service client.
2. Pull last 7 days (`startDate = today - 7d`, `endDate = today - 1d`) with `dimensions=["page", "query"]`, `rowLimit=25000`.
3. Pull prior 7 days (`startDate = today - 14d`, `endDate = today - 8d`) same dimensions.
4. Save current week to `gsc_snapshots` (append, date-stamped).
5. Join the two weeks on `(url, query)` in Python (or in SQL with a CTE).
6. Compute losers per row:
   - `position_change = curr_position - prev_position`
   - `impressions_change_pct = (curr_imp - prev_imp) / prev_imp * 100`
   - Severity classification:
     - `Critical`: prev_position ≤ 3 AND curr_position > 3
     - `High`: (prev_position ≤ 10 AND curr_position > 10) OR position_change ≥ 3
     - `Medium`: impressions_change_pct ≤ -30 AND prev_position ≤ 20
     - `Low`: position_change ≥ 1.5 OR impressions_change_pct ≤ -15
     - Else: skip (not a loser)
7. Insert losers into `query_losers` with `week_of = today`.
8. Rebuild `inventory` table: for each unique url, pick the row with the highest impressions, store as that url's `top_query` + stats. Set `section` using the Wellows Core-pattern regex (see below).

**Wellows Core-section URL patterns** (use these regexes to classify `section`):
```python
CORE_PATTERNS = [
    r'/pricing', r'/features', r'/product', r'/solutions',
    r'/geo\b', r'/aeo\b', r'/ai-visibility', r'/ai-citations',
    r'/chatgpt-', r'/perplexity-', r'/ai-overviews', r'/gemini-',
    r'/brand-visibility', r'/citation-'
]
def is_core(url: str) -> bool:
    return any(re.search(p, url, re.IGNORECASE) for p in CORE_PATTERNS)
```

### Job 3 — `find_link_suggestions` (Tuesday 04:00 UTC)

Replaces n8n "WF1 — Koray Linking Finder". This is the brain.

**Logic:**

1. Read `inventory`, `link_stats`, and `link_graph` from DB.
2. Build a `pages` dict keyed by URL, merging inventory + stats + (title, h1) from `link_graph` (use the most recent crawl entry for each source_url's title/h1).
3. Build `existing_links_set = {(source, target) for row in link_graph}` for deduplication.
4. **Identify receivers** (pages that almost rank): `position` in 4..20 AND `impressions` ≥ 30
5. **Identify donors** (pages with authority): `position` < 4 AND `clicks` > 5, OR `pagerank` > 0 AND `inbound_count` ≥ 2
6. For each (receiver, donor) pair where the link doesn't already exist:
   - **Anchor text** = receiver's H1 if available, else title, else top_query (Koray Lecture #74 — anchor MUST match target H1)
   - Skip if anchor < 5 chars
   - **Section link type** + weight (Koray section weighting):
     - `outer_to_core`: 1.5x — "Outer→Core: educational page links up to commercial cluster. Outer builds historical data, core drives monetization."
     - `core_to_core`: 1.2x — "Core→Core: sibling cluster reinforcement around central entity (AI visibility)."
     - `outer_to_outer`: 0.8x — "Outer→Outer: contextual bridge inside know-predicate territory."
     - `core_to_outer`: 0.4x — "Core→Outer: donates authority outward. Use sparingly per Koray Lecture #80."
   - **Topical overlap check**: split receiver's top_query into words > 3 chars, count how many appear in donor's (h1 + title + top_query). If overlap_ratio < 0.3, skip.
   - **Priority score** = `(impressions / position) × type_weight × overlap_ratio`
7. Sort by priority, take top 100.
8. For each of the top 100:
   - Fetch donor URL HTML (10-second timeout, log + skip on failure)
   - Call Claude Haiku 4.5 (`claude-haiku-4-5-20251001`) with this exact system + user prompt structure:

```python
system = """You are placing an internal link on a Wellows blog page following Koray Tugberk Gubur's semantic SEO rules. Wellows is an AI visibility SaaS that tracks brand citations across ChatGPT, Gemini, Perplexity, AI Overviews, and AI Mode.

Voice rules (non-negotiable):
- Conversational casual, plain vocabulary
- Confident, not hedgy
- No emojis, no GPT-style openers ("In today's...", "Let's dive in")
- Banned hype words: seamless, unlock, leverage, robust, cutting-edge, game-changer, supercharge
- Short sentences, no em-dash decoration

Koray rules:
1. Lecture #73: Place link AFTER the concept is defined. Never on the first word of a sentence.
2. Lecture #74: Anchor text must exactly match the target page's H1.
"""

user = f"""DONOR PAGE BODY (where the link will go):
{stripped_text[:6000]}

LINK TARGET:
- URL: {receiver_url}
- Target H1: {receiver_h1}
- Anchor text (must match H1): "{anchor_text}"

Return JSON only, no preamble:
{{
  "insertion_paragraph": "the existing paragraph from the donor page where the link fits naturally, with the anchor replaced by <a href=DUMMY>{anchor_text}</a>",
  "why_this_fits": "one sentence on why this placement satisfies Koray's rules"
}}"""
```

- Parse the response, replace `DUMMY` with `receiver_url` in `insertion_paragraph`.
- Upsert into `link_suggestions` with status='pending_review'.

**Rate limiting:** 5 concurrent Claude calls max (use `asyncio.Semaphore(5)`).

### Job 4 — `optimize_queued_urls` (Wednesday 05:00 UTC)

**Logic:**

1. Read `optimize_queue` WHERE status='optimize'.
2. For each URL:
   - Pull GSC last 90 days for THIS url only (use `dimensionFilterGroups` to filter by page)
   - Bucket queries by position: top3 (≤3), pos4_10, pos11_20, pos21plus
   - Fetch the page HTML, extract title + h1 + body text
   - Pull inbound + outbound links from `link_graph` for this URL
   - Take top 3 queries by impressions, call DataForSEO SERP API (`/v3/serp/google/organic/live/advanced`) — Basic auth with `DATAFORSEO_LOGIN:DATAFORSEO_PASSWORD`. For each query, extract top 5 organic results (rank, url, title, description).

## Web routes (FastAPI)

All routes behind HTTP Basic Auth using `ADMIN_PASSWORD` (single user, username = "admin").

### `GET /` — Dashboard home
- Top-level KPIs: total pages, total links, # orphans, # dead ends, # pending suggestions, # critical losers
- 4 cards linking to the four sub-views
- Last run timestamp for each job

### `GET /link-map` — Force-directed graph
- Server-side: read latest `link_stats` and `link_graph` from DB, render Jinja2 template
- Client-side: D3.js force-directed graph (use the Wellows brand colors below)
- Click a node → side panel shows inbound/outbound links, top GSC queries from `inventory`, section badge
- Filters: orphans only, dead-ends only, Core section only, search box

### `GET /suggestions` — Link Suggestions Inbox
- Table of `link_suggestions` WHERE status='pending_review', sorted by priority_score DESC
- Each row: donor_url, receiver_url, anchor_text, section_link_type badge, koray_rationale, insertion_sentence (as preview), priority_score
- Buttons per row: `Approve`, `Reject`, `Mark Inserted`
- `POST /suggestions/{id}/action` with `action` field updates status

### `GET /losers` — Query Loser Tracker
- Table of `query_losers` for the latest `week_of`, grouped by severity
- Columns: url (clickable to /optimize?url=...), query, prev_position → curr_position with delta, impressions delta
- "Send to Optimizer" button → adds row to `optimize_queue` with the URL pre-filled

### `GET /optimize` — URL Optimizer Queue
- Table of `optimize_queue` rows
- "Add URL" form: url, priority, notes → inserts new row with status='optimize'
- Each completed row shows a "View Brief" expand toggle that renders the markdown brief inline
- "Re-queue" button to set status back to 'optimize'

### `GET /api/link-graph.json` — Raw graph JSON
For the standalone D3 dashboard or any external consumer. Returns `{generated_at, nodes, edges}` matching the n8n format.

### `GET /healthz` — Health check (no auth)
Returns `{"status": "ok", "last_crawl": timestamp, "last_gsc": timestamp}`.

### `POST /jobs/{job_name}/run` — Manual job trigger
For testing without waiting for schedule. Accepts `crawl_link_map`, `gsc_inventory_and_losers`, `find_link_suggestions`, `optimize_queued_urls`.

## Critical implementation notes

- **GSC pagination**: when row count hits 25000, paginate with `startRow` and merge results before processing
- **Sitemap-index handling**: if `<sitemapindex>` is at the root, recursively fetch each child sitemap
- **PageRank**: use a simple dict-based implementation in `jobs/crawl_link_map.py`. Don't pull in `networkx` for this — adds 30MB to the image. 10 iterations is plenty for a 5000-page graph.
- **Claude rate limiting**: respect Anthropic's tier limits. For Tier 1 ($5 spend), that's 50 requests/min for Haiku — plenty for our use case. Add exponential backoff on 429.
- **Idempotency**: every job must be safe to re-run. Use upserts everywhere (`ON CONFLICT DO UPDATE` for inventory/link_stats/suggestions).
- **Logging**: use `loguru`, log to stdout (Replit captures this). Include job name + duration + row counts in every log line.
- **Graceful schedule timing**: jobs run Sunday→Monday→Tuesday→Wednesday in that order so each can read fresh data from the previous one.
